Data Safety & Privacy

✓ Client-Side Processing Only

How Your Data is Protected

Territory Sales Map employs a client-side processing architecture that prioritizes your data security and privacy. All file uploads and data processing occur directly within your web browser—no data is transmitted to or stored on our servers.

Data Processing

  • File uploads are parsed exclusively in your browser using XLSX.js and PapaParse
  • Geographic coordinates are matched locally using a reference database
  • Heatmap visualization is rendered in real-time on your device
  • Data export (CSV) is generated locally without server transmission

What Gets Transmitted

The only network requests made are:

  • A public ZIP code reference database (non-sensitive geocoding data)
  • Map tile imagery from OpenStreetMap CDN (map background only)

✓ Your customer data is never transmitted to any server

Data Retention

  • All imported data remains in your browser's memory during your session
  • No data is persisted on disk or servers
  • Closing your browser or navigating away clears all data
  • No cookies, tracking, or analytics of your data are collected

Recommended Usage

This application is designed for internal business use with trusted team members. While data never leaves your browser, we recommend:

  • Using Territory Sales Map within your organization's secure network
  • Limiting access to authorized personnel only
  • Avoiding use of shared or public computers for sensitive data
  • Clearing browser history after working with sensitive information

Technical Architecture

Territory Sales Map is built with a zero-server-processing model. Your browser becomes the exclusive processing environment, eliminating risks associated with cloud storage, data centers, or third-party vendors. This architecture ensures complete data sovereignty and privacy.

For questions about data privacy and security, please contact your system administrator.
